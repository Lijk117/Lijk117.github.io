Github 博客绑定域名

注：由于本博客才新建立，笔者也在学习过程中，还没有为图片配置图床，所以目前博客内能显示文字，是看不到图的，这个问题会在不久的将来解决，届时会更新此博客的以往内容，也会发文章讲如何配置图床。

如果你在浏览器输入这串数字 220.181.38.251
会发现进入了百度的首页，有没有觉得很…“神奇”？
这就是 ip 地址，其实每个网址背后都是 ip 地址，但这个太难记了，所以人们用了更简单的方式来记录网站 —— ”域名“，或者说：“网址”。

使用 github 注册的域名是 ”XXXX.github.io“，很长不好记，也不想带 “github.io”这个标，有没有别的办法能换成自己想要的网址？

当然能，只要把你想要用的网址和这个 域名网址进行匹配就行了，当有人输入你自己的网址，系统会自动引导到你的 github 博客网址上，简单说就是“来电转移”。

你可以试试 https://ljk.cool/
和 lijk117.github.io
这两个网址是不是都指向了同样的内容？

怎么做呢？

Enter 后，系统就会去检测你自己的网络和这个网址的延时，同时也能看到这个网址的 ip 地址，如下图：

用这个方法你可以去查任意网站的地址，都可以。

获取到这个 ip 地址后，在自己的域名解析的位置，要进行一下配置

按照下图配置：
注意哪里填 ip 地址，哪里填自己的原域名。

把这个配置完之后，还要去 github 上，把新旧域名进行绑定 —— 不是你随便建个网站，获取到别人的 ip 地址后，你就能够指引到别人家的，不行。

在 github 上找到自己的博客文件位置，点进去找到 “settings” - “Pages” - “Custom domain”，把自己的域名放上去，save，过一会儿就同步成功了。

在本地创建个文件，命名为 CHAME 注意全是大写。
在 _config.yml 上调整

url: http://ljk.cool
permalink: :year/:month/:day/:title/

注意放上自己的域名
CHAME 里面放自己的域名，然后保存在 public 文件夹里。
