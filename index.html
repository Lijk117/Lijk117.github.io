你需要一个云服务器来承担你的日常使用，成本大约 24 或者 32 块钱一个月，其实你买一个月也够了，只要你能够在一个月内完成你的博客搭建，就只需要三十来块钱就够了，当然，你可以一直续费，或者反复买云服务器。

下面开始。

购买云服务器

刚开始体验的时候，建议购买 “轻量应⽤服务器”，它相对来说⽐较便宜，海外的服务器只需要 24 /月 起，练习的时候可以使⽤，便宜。

  • 功能上都是一致的，这不必多说；

  • 价格上阿里云有优势，腾讯云的服务器是 32 块钱起了（海外的便宜），阿里云的云服务器是 24 块钱起。

无我

在讲“无我”之前必须要引入一些概念，这样才能大体上明白什么是“无我”。

缘起性空

我们常说的“缘分”就来源于此，所谓“缘起”：「诸法因缘生，诸法因缘灭。」诸法因为因缘的和合而生，因缘的消散而灭。
所谓性空，因为世间万物都是因缘合和而成，没有实在的“自体“，所以称为空。

举个例子：天上的云朵是由若干水分子组成的，这就是“缘起”，水分子动态聚合，可能变成云，也可能落下来变成雨，但它依然是云，这就是”性空”

山由土木草石构成，少了几棵草也是山，所以佛家有“看山不是山”的说法。

不管是“山”、“云”、“水”，它们都是个集合名词，那，“人”呢？

“我”呢？是由皮肤、血液、骨骼、内脏等构成的物质而已…

五蕴皆空

观自在菩萨行深般若波罗蜜多时，照见五蕴皆空，度一切苦厄。

观自在菩萨在发动般若智慧时候，看到了 五蕴皆空。
五蕴是指 色蕴、受蕴、想蕴、行蕴、识蕴。
色蕴是物质现象，其余四蕴是精神现象，有了上面缘起性空的铺垫，就会知道，物质世界的事物都是动态聚合飘忽无定的，那，构成这个世界的基础元素是什么？
中国人觉得是五行，印度人觉得是“四大” —— 地、水、火、风。
有了“四大”这基础元素，就可以知道我们的身体（物质）也是由这四大组成的，而这“四大”的行动并不以我的意志为转移，我不能多生出一个胳膊少去一条腿，甚至，我都不能主宰我的意识 ——
一方面很多事情都是在下意识下完成的，另一方面即便是明显有意识的活动，也是受限于自身的成长经历所做除的，最简单的…明知要好好学习，最终又去打游戏刷剧了…

一切物质形式和精神现象都是“蕴”，都是集合体。正因为都是集合体，所以变动分和没有自性，不由自主，这就是“空”。

我？无我

刚才提到说我不能主宰“我”的意识，也说了“我”是由“四大”构成的，但“四大”里任何一个元素都不是独立存在的，也不能够自主行动，它们都是在因果律给定的条件下，向着因果律方向上活动的，所以，“我”只不过是由一群基础元素构成的，在因果齿轮上不由自主运动的产物，这就是 —— 无我，“我”这个集合体不存在某个主宰。

有了以上的信息，再去看这个世界，山不再是山了，水也不再是水了，一切不过是由基础元素构成的没有自性的集合体罢了，而“我”与“你”，也不过是幻象。

去除“我执”

意识到这些，人的无穷烦恼就被解决得差不多了 —— 既然“无我”，又谈何“我的烦恼”呢？
人们对“我”太执着了 ——
别人听不懂“我”的话，我会非常生气；
“我”没得到怎样的结果，就会有不好的情绪；
”孩子“要是“我”的，不能是别人的。。。
很多人觉得私欲是万恶之源，朱熹主张“存天理，灭人欲”，佛学更彻底，连“我”都不存在了，更不会有分别心。
没有对“我”的执着，生活该有多快乐。

无我相，无人相，无众生相，无寿者相。
我们改变不了一切，包括我们自己 —— 甚至，“我”都不存在。

附一段《武林外传》的对话

最后

理解了“无我”的概念，并不是让你对任何事情都不在意，破罐子破摔，相反，它是让你能够更好地理解这个世界，控制自己的情绪，同时你要知道，这个世界确实不美好，但不能因此自暴自弃，而是积极寻求解脱。
“乐观厌世”是佛学的真谛，不然 —— 你将永受六道轮回之苦。

以上内容理解自 得到App 的《熊逸佛学五十讲》

加速国内 GitHub/GitBook 访问

由于一些特别原因，很多国外网站中国用户访问不了，或者访问非常慢，但这东西又是必须要用的，咋办？

可以这么做。

针对 mac 用户：

打开你的访达 ，同时按shift+command+G , 输入：

/private

之后，按 Enter, 进入 Private 文件夹

文件夹长下面这个样子，在 etc这个文件这里，点击右键，有个显示简介

你要按照下图的顺序，先把小锁头🔒解锁🔓，然后给 everyone 这个用户「读和写」权限，解锁需要密码，然后再把这个锁头给锁上。

这一步完成后，就双击进入到 etc 这个文件夹里，找到一个叫hosts的文件,也是右键点击显示简介，一样操作，也要给「读与写」权限，保存

使用 git 把内容推到 GitHub 上

  1. 先在github或者码云上创建一个公开或私有项目：例如demo
  2. 在本地也就是要推送的项目目录中使用 git init 进行初始化，把其变成git可以管理的仓库

  1. 若要忽略本地的文件或文件夹不被提交到git远程仓库 ，则需要在项目根目录下创建 .gitignore 文件
    touch .gitignore

  2. 打开文件，编辑内容，写如要忽略的文件，例如：
    .idea # python中的一个隐藏文件，自动生成的，不需要提交
    *.py[cod] #一些没有用的附带文件，直接忽略即可
    按照上诉操作后，保存，那么这些文件就不会被提交了。

  3. 将文件夹下所有文件从工作区保存到暂存区 git add .命令

  4. 把文件从暂存区提交到仓库区（本地的仓库） git-commit -m “此处写注释内容”

  5. 关联远程仓库 （第一次使用需要添加远程仓库的地址）

  6. 将远程库与本地同步（如果远程仓库里面有文件等内容需要执行以下这一步）
    git pull –rebase origin master

  7. 把本地内容推送到远程库 使用 git-push
    git push -u origin master
    按照上面的操作步骤就可以将本地文件推送到git远程仓库上了，并且可以自己设定不需要上传哪些文件。当然，情况因每人的配置环境或多或少出现问题，此教程只是解决众多问题中其中一个。

关于网络问题的事先说明

在写文档教程过程中，犯了一个严重的问题：

我默认了所有人的网络环境都跟我一样，都能打开。

因此，我忽略了倒不是 Github、GitBook 不能在国内使用，而是网络问题导致访问很慢很慢，需要等很久很久。

我自己在使用被限制的网络（即不开国外代理）的时候，真是好慢，慢到我非常绝望，我内心有很多方案：

  • 代注册？
  • 教翻墙？
  • 看看有没有替代的方案？
  • ···

我还是克制住了。首先我不能教翻墙，这东西自己用查资料没问题，提供服务是会被警告的，甚至可能罚款，这风险我不能担；

代注册确实最简单，但我能帮几个人注册？

替代方案我会继续找，目前找到一个方案，要 1999 一年，算了吧，继续找。

所以还是要自己想办法，也不是不能用，只是非常非常慢而已，忍一忍，多花些时间，等等吧，注册绑定这些东西弄完，也就好了。
